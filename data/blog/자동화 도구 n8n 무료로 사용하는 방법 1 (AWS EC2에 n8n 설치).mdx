저는 넷플릭스, 쿠팡, ChatGPT, Cursor, 노션 등등 너무 많은 것 같네요. 여러분도 분명 저와 같이 구독료를 지불하고 있는 서비스가 분명 많을 거라고 예상이 됩니다. 만약 이 강의를 수강하지 않으셨다면, 아마 여러분께서는 구독료 목록에 n8n 혹은 make와 같은 다른 자동화 도구도 추가가 되었을 것 같네요.

**n8n 자체 서버 호스팅 강의는 크게 2단계로 진행돼요. 딱 이 2단계만 진행하면 끝이에요. 🥳**

1. AWS EC2에 n8n 설치
2. 도메인 연동

이 강의에서는 n8n을 무료로 사용할 수 있도록 자체 서버로 호스팅하는 방법 중 첫번째 단계를 알려드릴게요. 제 강의만 따라하면 무리없이 자체 서버 호스팅을 완성할 수 있어요! 이 강의를 통해서 n8n 서버를 자체 호스팅을 구현하는 것도 좋지만, 더 나아가서 제가 알려드리는 개발 지식도 굉장히 값진 개념이기 때문에 가능하면 간단하게라도 보시는 것을 다시 한번 권장드려요.

## n8n 자체 서버 호스팅 개념

n8n을 무료로 사용하는 방법은 간단하게 우리가 직접 n8n이 동작하는 서버를 자체적으로 운영하면 돼요.

만약 예약 작업이나 조건 트리거 작업이 필요 없다면 자신의 컴퓨터에 n8n을 설치하여 자동화를 직접 수동으로 실행시키면 됩니다. 물론 당연히 비용은 들지 않고 이 방법은 굉장히 간단해요. 하지만 대부분의 자동화를 하려는 목적은 수동이 아닌 특정 시간에 자동으로 혹은 특정 액션이 발생했을 때 자동으로 운영하려는 목적들이 있죠. 그러기 위해서는 우리 컴퓨터가 아닌 24시간 운영되는 컴퓨터에 우리의 n8n을 설치하면 돼요.

**이렇게 24시간 운영되는 컴퓨터에 우리의 n8n을 설치하는 것이 바로 "n8n을 자체 서버에 호스팅 한다"라고 이해하면 돼요**. 그리고 24시간 운영되는 컴퓨터가 바로 AWS에서 제공해주는 EC2라는 서비스죠. n8n 자체 서버 호스팅을 하는 방법은 AWS EC2에 우리의 n8n을 설치하고 안전한 도메인 주소를 연동하면 돼요.

## AWS EC2에 n8n 설치 방법

1. AWS 계정 만들기
2. EC2 생성 및 설정하기
3. EC2에 n8n 설치하기

아무리 어렵고 복잡하더라도 제가 알려드리는 방법을 그대로 따라하시면 n8n을 무료로 사용하실 수 있으니 천천히 따라하시기 바랍니다. **위에서 설명했듯이 EC2는 24시간 운영되는 컴퓨터라고 생각하시면 돼요.**

자 그럼 굳이 시작해볼까요?

### 1. AWS 계정 만들기

#### 1.1 AWS 가입하여 프리티어 계정 만들기

가장 먼저 AWS에 가입을 해요. 새로 가입하는 계정은 1년 동안 무료로 사용할 수 있는 프리티어 계정을 발급 받아요. 그러니 가능한 AWS 계정이 있더라도 새로이 가입하는 것을 권장드려요. 기존 계정을 사용하더라도 서버비는 1달에 2달러 내외이니 가입에 문제가 있으면 기존 계정을 사용하셔도 좋아요.

### 2. EC2 생성 및 설정하기

#### 2.1 EC2 생성하기

자 이제 우리가 사용할 24시간 운영되는 컴퓨터를 생성해볼까요?

1. AWS에서 EC2 서비스를 검색하여 EC2 대시보드 페이지로 이동합니다.
2. 왼쪽 메뉴에서 인스턴스 메뉴를 클릭하여 인스터스 페이지로 이동합니다.
3. 인스턴스 페이지에서 오른쪽 상단에 있는 인스턴스 시작 버튼을 클릭하여 인스턴스 생성 페이지로 이동합니다.
4. 원하시는 이름을 입력하고, 애플리케이션 및 OS 이미지 영역에서 Amazon Linux로 설정된 환경을 Ubuntu로 변경합니다.
5. 키 페어 영역에서 새 키 페어 생성을 하여 .pem 파일을 다운받습니다.
6. 네트워크 설정 영역에서 보안 그룹 생성을 선택하고 SSH, HTTPS, HTTP 트래픽 허용을 체크합니다.
7. 마지막으로 인스턴스 생성 버튼을 클릭하여 생성을 시작합니다. (약 1분 ~ 2분)

#### 2.1 고정 IP 설정하기

위에서 생성한 우리의 EC2의 IP를 고정시켜서 항상 동일한 IP 주소로 동작하게 설정해볼까요?

1. EC2 콘솔에서 네트워크 및 보안 메뉴에 있는 탄력적 IP 메뉴를 클릭하여 탄력적 IP 페이지로 이동합니다.
2. 우측 상단에 있는 새 IP를 생성하여 우리가 만들어 놓은 EC2 인스턴스에 연결을 합니다.
3. 연결된 인스턴스 ID에 우리의 EC2가 연결되었으면 설정이 완료된 겁니다.

#### 2.3 EC2 접속하기

이제 한번 우리가 생성한 EC2에 가장 간단한 방법으로 접속을 해볼까요?

1. EC2 콘솔에서 인스턴스 메뉴를 눌러 인스턴스 페이지로 이동합니다.
2. 접속할 인스턴스의 인스턴스 ID를 클릭하여 해당 인스턴스 페이지로 이동합니다.
3. 오른쪽 상단에 있는 연결 버튼을 클릭하여 인스턴스에 연결 페이지로 이동합니다.
4. 인스턴스에 연결 페이지에서 첫번째 탭 메뉴인 EC2 인스턴스 연결 영역에서 오른쪽 하단에 있는 연결 버튼을 클릭합니다.
5. 검은 화면에 하얀 영어로 쓰여있는 페이지가 뜨면 접속이 된겁니다.

### 3. EC2에 n8n 설치하기

#### 3.1 EC2에 n8n 설치하기

EC2에 접속하면 검은 화면에 영어만 있지만, 이것도 사실 정말 간단한 거니 천천히 따라오시기 바랍니다. 검은 화면에 아래의 내용을 정확하게 적어주시기 바랍니다.

1. `curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -`
   - n8n을 실행할 수 있는 nodejs를 설치할 수 있는 환경을 준비하는 명령어 입니다.
2. `sudo apt install -y nodejs`
   - n8n을 실행할 수 있는 nodejs를 설치하는 명령어 입니다. 약 5분 정도 소요되니 기다려주세요.
3. `sudo npm install -g n8n`
   - n8n을 설치하는 명령어 입니다.
